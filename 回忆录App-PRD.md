# 老年人回忆录写作App - 产品需求文档 (PRD)

**产品名称**：人生回忆录 / LifeMemoir  
**目标用户**：60岁以上老年人（及希望为父母记录人生的子女）  
**产品愿景**：让每一位老人都能轻松留下自己的人生故事，成为家族传承的珍贵礼物

---

## 1. 产品概述

### 1.1 核心价值主张
- **零门槛**：不识字的老人也能通过语音完成回忆录
- **智能化**：AI像老朋友一样引导、追问、整理
- **多形态**：从口语到白描，再到多种作家风格
- **可传承**：生成精美实体书/电子书/有声书

### 1.2 用户画像

**主要用户：老年用户（张阿姨，68岁）**
- 小学文化，不擅长打字
- 视力不太好，需要大字体
- 普通话带方言口音
- 有很多人生故事想说但不会写
- 希望给孙辈留下点什么

**次要用户：子女用户（小李，35岁）**
- 想帮妈妈记录人生故事
- 工作忙碌，没时间长时间陪伴
- 希望成品精美，可传家珍藏

## 2. 功能需求

### 2.1 AI引导式访谈系统

#### 2.1.1 人生阶段划分
| 阶段 | 年龄段 | 主题 | 预计问题数 |
|------|--------|------|-----------|
| 童年时光 | 0-12岁 | 家庭、学校、游戏、过年 | 15-20 |
| 青春岁月 | 13-25岁 | 求学、初恋、梦想、第一份工作 | 15-20 |
| 成年生活 | 26-60岁 | 婚姻、育儿、工作、朋友 | 20-25 |
| 晚年感悟 | 60岁+ | 智慧、遗憾、寄语、总结 | 10-15 |

#### 2.1.2 问题设计原则
1. **开放式**：避免是/否问题
2. **具体化**：从具体场景切入，而非抽象概念
3. **情感化**：关注感受和心情，不只是事实
4. **渐进式**：从简单到深入，建立信任

#### 2.1.3 AI动态调整策略
- 根据回答长度判断老人状态（疲劳/兴奋）
- 根据情绪关键词调整问题深度
- 发现故事线索时自动深入追问
- 老人卡壳时提供温和提示

### 2.2 语音交互系统

#### 2.2.1 语音朗读 (TTS)
- **语速调节**：0.5x - 1.5x（默认0.8x适合老人）
- **方言支持**：普通话、粤语、四川话、上海话等
- **音色选择**：温和女声（默认）、亲切男声
- **断句优化**：长句自动分段，重点词停顿

#### 2.2.2 语音输入 (ASR)
- **方言识别**：支持主要方言口音
- **降噪处理**：过滤环境噪音
- **实时转写**：边说边显示文字
- **确认机制**：转写后让老人确认或重说

#### 2.2.3 语音控制指令
| 指令 | 功能 |
|------|------|
| "开始" / "开始吧" | 开始访谈 |
| "暂停" / "等一下" | 暂停当前问题 |
| "重听" / "再说一遍" | 重复当前问题 |
| "下一个" / "过" | 跳过当前问题 |
| "说完了" / "好了" | 结束当前回答 |
| "保存" / "存起来" | 保存当前进度 |

### 2.3 5W1H智能追问系统

#### 2.3.1 追问触发条件
当老人回答出现以下情况时自动追问：
- 时间模糊（"那时候"、"很久以前"）→ 追问When
- 地点缺失 → 追问Where
- 人物不清 → 追问Who
- 事件笼统 → 追问What
- 原因未提 → 追问Why
- 过程简略 → 追问How

#### 2.3.2 追问语气设计
- **温和型**："您刚才提到那时候，大概是什么时候呢？"
- **好奇型**："听起来很有意思，当时还有谁在场呢？"
- **共情型**："那一定很不容易，您当时是怎么挺过来的？"

#### 2.3.3 追问深度控制
- 每个主题最多追问3层
- 老人表现出疲劳时停止追问
- 老人情绪激动时转向安慰

### 2.4 时间线整理系统

#### 2.4.1 故事结构化
- 自动提取时间关键词（年份、季节、年龄段）
- 识别人物关系网络
- 标记关键事件和转折点
- 发现故事间的因果关系

#### 2.4.2 章节自动生成
```
第一章：童年时光（1945-1957）
  1.1 家乡与家庭
  1.2 小学岁月
  1.3 童年玩伴

第二章：青春岁月（1958-1970）
  2.1 求学之路
  2.2 初恋故事
  2.3 第一份工作
...
```

#### 2.4.3 可视化时间线
- 横向时间轴展示人生大事
- 可点击查看详情
- 支持缩放（年/月/日）

### 2.5 AI润色系统

#### 2.5.1 润色规则
| 问题类型 | 处理方式 | 示例 |
|----------|----------|------|
| 语气词 | 删除或替换 | "那个...就是..." → 删除 |
| 重复 | 去重 | "我我我当时" → "我当时" |
| 口误 | 智能纠正 | "我妹妹...哦不，我姐姐" → "我姐姐" |
| 倒装 | 调整语序 | "饭吃过了我" → "我吃过了饭" |
| 省略 | 适当补全 | "然后...就那样了" → 根据上下文补全 |

#### 2.5.2 保持原貌原则
- 保留老人的语言习惯和口头禅
- 保留方言特色和时代用语
- 保留情感表达和感叹词
- 只修正影响理解的错误

### 2.6 作家风格系统

#### 2.6.1 预设风格
| 风格 | 特点 | 适用场景 |
|------|------|----------|
| 白描纪实 | 朴素真实，原汁原味 | 默认版本 |
| 沈从文 | 质朴温情，乡土气息 | 农村背景 |
| 张爱玲 | 细腻苍凉，都市感 | 城市女性 |
| 汪曾祺 | 平淡闲适，人情味 | 知识分子 |
| 余华 | 简洁深刻，时代感 | 经历过苦难 |
| 杨绛 | 温婉智慧，从容 | 高龄女性 |

#### 2.6.2 风格迁移技术
- 基于大模型的风格迁移
- 保留事实内容，改变表达方式
- 支持自定义风格（上传样本学习）

### 2.7 导出与成书系统

#### 2.7.1 导出格式
- **PDF**：适合打印，多种排版模板
- **EPUB**：电子书，支持手机/平板阅读
- **Word**：可继续编辑
- **音频**：AI朗读的有声书（MP3/M4B）

#### 2.7.2 排版模板
- 经典版：传统书籍排版
- 现代版：简洁清新风格
- 相册版：图文混排，适合插入照片
- 家谱版：包含家族树和时间线

#### 2.7.3 实体书印刷
- 对接印刷服务商
- 支持精装/平装
- 可选封面材质
- 预计价格：50-200元/本

## 3. 用户体验设计

### 3.1 界面设计原则

#### 3.1.1 视觉设计
- **大字体**：正文18px+，标题24px+
- **高对比**：深色文字+浅色背景
- **简洁布局**：每屏只有一个主要操作
- **温暖色调**：米黄、浅棕、淡绿等舒适颜色

#### 3.1.2 交互设计
- **大按钮**：点击区域不小于60x60px
- **语音优先**：所有文字都有语音版
- **进度可见**：显示完成进度和鼓励
- **容错设计**：误操作可轻松撤销

### 3.2 使用流程

```
启动App → 欢迎语（语音）
    ↓
创建回忆录 → 输入基本信息（可由子女代填）
    ↓
选择阶段 → 童年/青春/成年/晚年（可跳选）
    ↓
开始访谈 → AI提问 → 语音回答 → 智能追问
    ↓
（循环直到该阶段完成）
    ↓
查看整理 → 时间线浏览 → 补充修改
    ↓
选择风格 → 白描/沈从文/张爱玲...
    ↓
生成预览 → 查看效果 → 调整
    ↓
导出成书 → PDF/实体书/有声书
```

### 3.3 情感设计

#### 3.3.1 AI人设
- **名字**：小忆（亲切好记）
- **性格**：耐心、温柔、好奇、尊重
- **说话方式**：像孙女/孙子一样亲切
- **不会说的**："快点"、"这么简单"、"你不懂"

#### 3.3.2 鼓励机制
- 完成每个阶段："您讲得真好！这段故事太珍贵了！"
- 回答精彩时："哇，这个细节太有意思了！"
- 老人疲劳时："咱们休息一下吧，不着急"
- 完成回忆录："恭喜您！这是给家人的珍贵礼物！"

## 4. 技术需求

### 4.1 核心技术栈
- **前端**：React Native（跨平台）
- **后端**：Python + FastAPI
- **AI模型**：GPT-4 / Claude（对话+润色+风格迁移）
- **语音**：Azure Speech / 科大讯飞（TTS+ASR）
- **数据库**：PostgreSQL + Redis
- **存储**：OSS（音频、图片、导出文件）

### 4.2 性能要求
- 语音转写延迟 < 2秒
- AI响应延迟 < 3秒
- 支持离线模式（先录音，联网后转写）
- 导出10万字PDF < 30秒

### 4.3 安全隐私
- 所有数据加密存储
- 本地优先，云端同步可选
- 老人可完全删除自己的数据
- 不向第三方出售数据
