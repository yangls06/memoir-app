# AI记事本 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 一句话描述
AI记事本是一款智能笔记管理应用，结合传统笔记功能与AI能力，帮助用户高效记录、整理和回顾信息。

### 1.2 目标用户
- **主要用户**: 知识工作者、学生、研究人员
- **次要用户**: 需要记录日常事务的普通用户
- **用户画像**: 
  - 经常需要记录学习笔记、会议记录、想法灵感
  - 希望快速找到历史笔记内容
  - 希望AI辅助整理和总结大量信息

### 1.3 核心价值主张
- **记录简单**: 支持Markdown，专注写作体验
- **智能整理**: AI自动总结，快速把握笔记要点
- **高效检索**: 智能搜索，快速定位所需信息
- **问答助手**: 基于笔记内容，AI回答用户问题

## 2. 详细功能需求

### 2.1 核心功能

#### F1: 笔记CRUD操作
**功能描述**: 用户可以创建、查看、编辑、删除笔记

**验收标准**:
- [ ] 用户可以创建新笔记，输入标题和内容
- [ ] 用户可以查看笔记列表，按创建时间倒序排列
- [ ] 用户可以点击笔记进入编辑模式
- [ ] 用户可以删除笔记，删除前需确认
- [ ] 笔记自动保存草稿，防止意外丢失
- [ ] 支持笔记置顶功能

#### F2: Markdown编辑器
**功能描述**: 笔记内容支持Markdown格式编辑和实时预览

**验收标准**:
- [ ] 支持标准Markdown语法（标题、列表、链接、图片、代码块等）
- [ ] 提供编辑/预览双栏模式
- [ ] 提供常用格式快捷按钮（加粗、斜体、列表等）
- [ ] 支持代码块语法高亮
- [ ] 支持表格编辑
- [ ] 支持数学公式（LaTeX）

#### F3: AI自动总结
**功能描述**: AI自动分析笔记内容并生成摘要

**验收标准**:
- [ ] 用户可一键生成笔记摘要
- [ ] 摘要长度可配置（简短/中等/详细）
- [ ] 摘要自动保存，显示在笔记顶部或侧边
- [ ] 支持批量总结多篇笔记
- [ ] 摘要更新时保留历史版本

#### F4: AI问答助手
**功能描述**: 用户可基于笔记内容向AI提问

**验收标准**:
- [ ] 提供聊天界面，用户可输入问题
- [ ] AI基于用户所有笔记或指定笔记回答问题
- [ ] 回答中引用相关笔记来源
- [ ] 支持上下文连续对话
- [ ] 可指定搜索范围（单篇/多篇/全部笔记）

#### F5: 笔记分类与标签
**功能描述**: 支持文件夹分类和标签管理

**验收标准**:
- [ ] 支持创建文件夹/笔记本，层级不限
- [ ] 支持为笔记添加多个标签
- [ ] 标签支持颜色区分
- [ ] 支持按文件夹/标签筛选笔记
- [ ] 支持快捷标签输入（自动补全）

#### F6: 笔记搜索
**功能描述**: 支持全文搜索和智能搜索

**验收标准**:
- [ ] 支持标题和内容全文搜索
- [ ] 搜索结果高亮显示关键词
- [ ] 支持按时间范围筛选
- [ ] 支持按标签/文件夹筛选
- [ ] 搜索支持模糊匹配
- [ ] 保存搜索历史

---

### 2.2 次要功能

#### F7: 笔记导入导出
**功能描述**: 支持多种格式的导入导出

**验收标准**:
- [ ] 支持导出为Markdown文件
- [ ] 支持导出为PDF
- [ ] 支持批量导出
- [ ] 支持导入Markdown文件
- [ ] 支持从其他笔记软件导入（可选）

#### F8: 笔记分享
**功能描述**: 支持分享笔记给他人

**验收标准**:
- [ ] 生成分享链接
- [ ] 可设置链接有效期
- [ ] 可设置访问密码（可选）
- [ ] 支持取消分享

#### F9: 回收站
**功能描述**: 删除的笔记进入回收站，支持恢复

**验收标准**:
- [ ] 删除笔记进入回收站
- [ ] 回收站支持恢复笔记到原位置
- [ ] 回收站支持彻底删除
- [ ] 自动清理30天前的回收站内容

#### F10: 快捷键支持
**功能描述**: 提供常用操作的键盘快捷键

**验收标准**:
- [ ] 新建笔记快捷键（Ctrl/Cmd + N）
- [ ] 保存笔记快捷键（Ctrl/Cmd + S）
- [ ] 搜索快捷键（Ctrl/Cmd + K）
- [ ] 快捷键可自定义（可选）

## 3. 用户故事

### US1: 创建笔记
> 作为一位学生，
> 我希望快速创建一篇新的学习笔记，
> 以便记录课堂上的重要知识点。

**验收标准**: 3秒内完成新笔记创建并开始编辑

### US2: Markdown编辑
> 作为一位程序员，
> 我希望使用Markdown格式编写技术笔记，
> 以便插入代码块和格式化文本。

**验收标准**: 支持代码高亮和实时预览

### US3: AI总结
> 作为一位研究员，
> 我希望AI帮我总结长篇笔记的核心观点，
> 以便快速回顾内容要点。

**验收标准**: 总结准确率达到80%以上，生成时间<5秒

### US4: AI问答
> 作为一位项目经理，
> 我希望向AI询问之前会议记录中的决策内容，
> 以便快速找到相关信息而不用逐篇翻阅。

**验收标准**: AI能准确回答基于笔记内容的问题

### US5: 分类管理
> 作为一位多项目并行的工作者，
> 我希望将笔记按项目分类并添加标签，
> 以便有条理地管理不同领域的笔记。

**验收标准**: 支持多级文件夹和多个标签

### US6: 搜索笔记
> 作为一位有大量笔记的用户，
> 我希望通过关键词快速找到需要的笔记，
> 以便节省查找时间。

**验收标准**: 搜索结果在1秒内返回，支持模糊匹配

### US7: 跨设备访问
> 作为一位经常切换设备的用户，
> 我希望在不同设备上访问我的笔记，
> 以便随时随地查看和编辑。

**验收标准**: 数据实时同步，支持Web端访问

## 4. 非功能需求

### 4.1 性能要求
| 指标 | 要求 | 说明 |
|------|------|------|
| 页面加载时间 | < 2秒 | 首屏加载完成 |
| 笔记保存延迟 | < 500ms | 自动保存反馈 |
| 搜索结果返回 | < 1秒 | 千级笔记量 |
| AI总结生成 | < 5秒 | 单篇5000字以内 |
| AI问答响应 | < 3秒 | 首次响应时间 |
| 并发用户支持 | 1000+ | 同时在线用户 |

### 4.2 安全要求
- 用户数据加密存储（AES-256）
- 传输层加密（HTTPS/TLS）
- 用户认证：JWT Token机制
- 敏感操作需二次确认
- AI交互数据脱敏处理
- 定期数据备份

### 4.3 兼容性要求
- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **移动端**: 响应式设计，支持手机/平板访问
- **操作系统**: Windows, macOS, Linux, iOS, Android

### 4.4 可用性要求
- 系统可用性 > 99.9%
- 支持离线编辑（本地缓存）
- 数据自动同步，冲突自动解决
- 提供数据导出功能（防止锁定）

### 4.5 可扩展性要求
- 支持插件/扩展机制（未来）
- API接口设计预留扩展空间
- 支持第三方集成（可选）

## 6. MVP范围定义

### 6.1 MVP目标
在4周内交付一个可用的AI记事本产品，包含核心笔记功能和AI能力。

### 6.2 MVP功能清单

#### 必须实现 (P0)
1. ✅ 用户注册/登录
2. ✅ 笔记的创建、编辑、删除
3. ✅ Markdown编辑器（基础语法）
4. ✅ AI总结功能
5. ✅ AI问答功能
6. ✅ 文件夹分类
7. ✅ 标签管理
8. ✅ 全文搜索

#### 明确排除 (不在MVP)
- 团队协作/实时协作
- 笔记分享功能
- 移动端App（仅Web响应式）
- 第三方集成
- 高级权限管理
- 笔记历史版本

### 6.3 MVP验收标准
- [ ] 用户可以完成完整的笔记生命周期（创建→编辑→查看→删除）
- [ ] Markdown渲染正确，支持常见语法
- [ ] AI总结功能可用，生成合理的摘要
- [ ] AI问答能基于笔记内容正确回答问题
- [ ] 搜索功能能在1秒内返回结果
- [ ] 系统稳定运行，无严重Bug

### 6.4 成功指标
| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 用户注册数 | 100+ | 首月 |
| 笔记创建数 | 500+ | 首月累计 |
| AI功能使用率 | > 60% | 活跃用户中 |
| 用户留存率 | > 30% | 7日留存 |
| 系统稳定性 | > 99% | 在线率 |
